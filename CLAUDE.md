# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

Package manager is `pnpm`. Do not use `npm` or `yarn` — it will fight the pnpm lockfile.

- `pnpm dev` — start local dev server (Astro default port)
- `pnpm build` — build static site to `./dist/`
- `pnpm preview` — preview the built site
- `pnpm astro check` — type-check `.astro` files and content collections
- `pnpm astro add <integration>` — add an official Astro integration

No test runner or linter is configured. Prettier (with `prettier-plugin-astro`) is the only formatting tool — run via `pnpm exec prettier --write <path>`.

Astro 6 requires Node >=22.12. Use `nvm use --lts` if the local default is older.

## Architecture

Static Astro 6 portfolio site. No framework integrations (React, Vue, etc.) — pure `.astro` components + a single `global.css`. TypeScript uses `astro/tsconfigs/strict`. Deploy target is Netlify (static output, no adapter configured).

### Content collections

`src/content.config.ts` defines a single `work` collection using the Content Layer `glob` loader over `src/content/work/**/*.md`. Each portfolio project is a Markdown file with this frontmatter schema:

```
title, description, publishDate (date), tags (string[]), img (string), img_alt?, link? (url), github? (url)
```

Pages for each work entry are generated by `src/pages/work/[...slug].astro` via `getStaticPaths()` over `getCollection("work")`. The route param is `entry.id` (Content Layer), and page content is rendered via the top-level `render(entry)` helper from `astro:content`. Adding a new project = drop a new `.md` file in `src/content/work/` — no code changes needed.

### Layout and theming

All pages wrap content in `src/layouts/BaseLayout.astro`, which renders `MainHead`, `Nav`, a `<slot />`, and `Footer`. The layout owns the background-image system: it declares CSS custom properties (`--bg-image-*`) that reference assets in `public/assets/backgrounds/`, with light/dark variants swapped via the `theme-dark` root class. Below-the-fold backgrounds are lazy-loaded by toggling `.loaded` on `<html>` after `window.load`. Dark mode is driven by `ThemeToggle.astro`.

Global design tokens (colors, gradients, typography scale, spacing, shadows) live in `src/styles/global.css`. Component styles are scoped `<style>` blocks inside each `.astro` file and reference these tokens via `var(--...)`.

### Images

Images referenced from Markdown frontmatter (e.g. `/assets/portfolio.webp`) resolve against `public/`. The project is migrating raster assets to `.webp` — when adding new images prefer `.webp` and update any existing frontmatter `img` paths together with the asset.
